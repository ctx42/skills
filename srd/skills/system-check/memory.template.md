# System Memory

Curated knowledge base for `system-check`. One atomic fact per line, grouped
by topic. Each line ends with its source in brackets — a corpus document id
when a documentation-corpus MCP server is used, else a path relative to the
space root below; `[unwritten]` marks tribal facts in no doc, `[same]` repeats
the line above. The skill maintains this file; it grows as SRD reviews surface
facts.

Space root: `<unset>` — set this to your local documentation root. Leave it
`<unset>` if a corpus MCP server is available (it is preferred and needs no
root) or if you have no space docs; the skill then runs on the corpus alone, or
without the live-doc layer using only the facts below.
